# Guide: Clean System Script

This guide provides instructions on how to use the `clean-system.sh` script, a system-wide cache, temp, and junk file cleanup utility.

## Overview

The `clean-system.sh` script cleans disposable files across the system to reclaim disk space. It targets both user-space locations (no root required) and system-level caches (root required).

The script performs:
- **User Cache Cleanup**: Removes files in `~/.cache/` older than a configurable threshold
- **Thumbnail Cache Cleanup**: Removes all thumbnail images (auto-regenerated by the system)
- **Trash Emptying**: Empties `~/.local/share/Trash/`
- **Temp File Cleanup**: Removes stale temp files from `/tmp/` and `~/.local/state/`
- **DNF Cache Cleanup**: Cleans the DNF package manager cache
- **Journal Vacuum**: Trims systemd journal logs to a retention period
- **Container Prune**: Removes unused podman (preferred) or Docker images, containers, and build cache

All operations support dry-run mode for safe preview.

## Dependencies

Most targets use standard system utilities. Optional targets require:

```bash
# Standard (usually pre-installed)
sudo dnf install coreutils findutils

# Optional: for --dnf target
sudo dnf install dnf

# Optional: for --journal target
# (journalctl is part of systemd, pre-installed on Fedora)

# Optional: for --containers target (podman is default on Fedora)
sudo dnf install podman
# Or, alternatively:
sudo dnf install docker
```

## Installation & Setup

1. Make the script executable:
   ```bash
   chmod +x scripts/maintenance/clean-system.sh
   ```

2. (Optional) Create an alias in `~/.bashrc`:
   ```bash
   alias cleansys='sudo bash "$HOME/Documents/code/fedora-user-scripts/scripts/maintenance/clean-system.sh"'
   source ~/.bashrc
   ```

## Usage

### Run All Targets

```bash
# With sudo (includes system-level targets)
sudo ./scripts/maintenance/clean-system.sh

# Without sudo (user-space only, system targets are skipped)
./scripts/maintenance/clean-system.sh
```

### Preview Everything (Dry Run)

```bash
./scripts/maintenance/clean-system.sh --dry-run
```

### User-Space Only

```bash
./scripts/maintenance/clean-system.sh --cache --thumbnails --trash --temp
```

### System-Level Only

```bash
sudo ./scripts/maintenance/clean-system.sh --dnf --journal --containers
```

### Individual Targets

```bash
# Only clean user cache
./scripts/maintenance/clean-system.sh --cache

# Only empty trash
./scripts/maintenance/clean-system.sh --trash

# Only prune containers (auto-detects podman or docker)
sudo ./scripts/maintenance/clean-system.sh --containers
```

## Configuration

Set these in `~/.config/fedora-user-scripts/config.sh`:

```bash
# Cache file age threshold in days (default: 30)
CLEAN_CACHE_DAYS=30

# Temp file age threshold in days (default: 7)
CLEAN_TEMP_DAYS=7

# Journal log retention in days (default: 7)
CLEAN_JOURNAL_DAYS=7
```

## Log Files

When run with sudo, the script creates logs in `/var/log/`:
- Log file: `/var/log/clean-system-YYYYMMDD-HHMMSS.log`
- Permissions: `600` (readable only by root)

## Troubleshooting

### "Skipping ... (requires sudo)"
Run the script with `sudo` to include system-level targets, or run them separately:
```bash
sudo ./scripts/maintenance/clean-system.sh --dnf --journal --containers
```

### "Docker daemon not running, skipping"
This only appears if you have Docker installed but the daemon is stopped. If you use podman instead, this message won't appear.
```bash
sudo systemctl start docker
```

### Cache still large after cleanup
The default age threshold is 30 days. Reduce it to clean more aggressively:
```bash
# In ~/.config/fedora-user-scripts/config.sh
CLEAN_CACHE_DAYS=7
```

### Thumbnails reappear after cleaning
This is expected. Thumbnails are regenerated by the file manager when you browse directories. The cache will grow again over time.

## Best Practices

- Run with `--dry-run` first to preview what will be removed
- Run periodically (e.g., weekly or monthly) to prevent cache buildup
- Use `sudo` for full cleanup including system-level targets
- Adjust age thresholds in config for your usage patterns
- Combine with `fedora-update.sh` for comprehensive monthly maintenance
